Content Description

This lecture focuses on three of the main R data types: vectors, matrices and arrays. These are R data objects with specific properties and with a direct link to algebraic operations. In most cases they will be numerical, but they can also be of character or logical class. All of them are restricted to a single class of elements, i.e. being exclusively numeric or coersed to character if different classes are introduced.

VECTORS

A vector is a simple array (one dimmension) of numbers or characters, such as the measurements of a single variable A vector is a sequence of data elements of the same basic class A vector is defined by its mode and length

We already saw yesterday how the combine function can be used to create vectors.

# a numeric vector
x <- c(20, 50, 100)  
x
## [1]  20  50 100
# a logical vector
x <- c(TRUE, FALSE, TRUE, FALSE, FALSE)
x
## [1]  TRUE FALSE  TRUE FALSE FALSE
# a vector of characters
x <- c("GG","TT","GA","AT","CC")
x
## [1] "GG" "TT" "GA" "AT" "CC"

Note that a vector cannot mix different types of elements. In doing so, numbers are coerced to characters to maintain the same type of elements.

x <- c(2,3,5)
y <- c("a", "b", "c")
z <- c(x,y)
z
## [1] "2" "3" "5" "a" "b" "c"
class(x)
## [1] "numeric"
class(y)
## [1] "character"
class(z)
## [1] "character"

A vector can be defined by its mode() and its lenght()

mode(x)
length(x)

Naming vectors

The elements of a vector can be named. This is useful for data organization and inndexing.

size1 <- c(20, 50, 100, 25 , 5, 20)

names(size1) <- country

OPERATIONS WITH VECTORS

Operations with vectors are conducted elementwise.

a <- c(10, 30, 50, 70, 25)
5 + a
## [1] 15 35 55 75 30
5 * a
## [1]  50 150 250 350 125
a / 5
## [1]  2  6 10 14  5
b <- seq(5, 25, by=5)
c <- a + b
a
## [1] 10 30 50 70 25
b
## [1]  5 10 15 20 25
c
## [1] 15 40 65 90 50

Recycling

When two vectors are of unequal length, the shorter one will be recycled in order to match the longer vector

d <- c(20, 5)
a - d
## Warning in a - d: longer object length is not a multiple of shorter object
## length
## [1] -10  25  30  65   5
a * d
## Warning in a * d: longer object length is not a multiple of shorter object
## length
## [1]  200  150 1000  350  500

VERY IMPORTANT: READ WARNINGS AND ERRORS!!! But be careful, because the warning is only shown if the vectors are fractionally recycled:

length(a)
## [1] 5
length(d)
## [1] 2
rep(a, 2) - d
##  [1] -10  25  30  65   5   5  10  45  50  20

Comparing vectors

R uses specific symbology that allows us to compare vectors, again in an elemntwise manner.

size2 <- c(12, 15, 10, 5, 5, 15)

## [1] FALSE FALSE FALSE FALSE  TRUE FALSE

size1%in%size2

MATRICES

A matrix is a two-dimensional array (two dimmensions) of values A matrix is a collection of vectors (e.g. measurements of several variables for the same objects) A matrix can only contain data of the same type, as is the case for vectors

Let’s define three vectors: V1 containing numbers form 1 to 5 V2 containing numbers form 100 to 108 with a step of 2 V3 containing the number 50 five times

V1 <- c(1:5)
V2 <- seq(100, 108, 2)
V3 <- rep(50, 5)

Creating matrices

Two major functions for creating matrices are cbind() and rbind(), which allow us to bind together vectors (or matrices) by columns or by rows.

cbind(V1, V2, V3)
##      V1  V2 V3
## [1,]  1 100 50
## [2,]  2 102 50
## [3,]  3 104 50
## [4,]  4 106 50
## [5,]  5 108 50
rbind(V1, V2, V3)
##    [,1] [,2] [,3] [,4] [,5]
## V1    1    2    3    4    5
## V2  100  102  104  106  108
## V3   50   50   50   50   50

This works because the three vectors have the same length, but if that were not the case R would apply recycling to bind the vectors.

V4 <- 10:16
rbind(V1, V4)
## Warning in rbind(V1, V4): number of columns of result is not a multiple of
## vector length (arg 1)
##    [,1] [,2] [,3] [,4] [,5] [,6] [,7]
## V1    1    2    3    4    5    1    2
## V4   10   11   12   13   14   15   16

Again, be careful, because the warning is shown only if the vectors are fractionally recycled:

V5 <- 1:3
V6 <- 1:6
rbind(V5,V6)
##    [,1] [,2] [,3] [,4] [,5] [,6]
## V5    1    2    3    1    2    3
## V6    1    2    3    4    5    6

One can also perform summary operations by rows or by columns of a matrix:

colSums(mat1)
## [1] 151 154 157 160 163
rowSums(mat1)
##  V1  V2  V3 
##  15 520 250
colMeans(mat1)
## [1] 50.33333 51.33333 52.33333 53.33333 54.33333
rowMeans(mat1)
##  V1  V2  V3 
##   3 104  50

A more general function that allows us to do this - and many many other things - is apply(). Apply uses the margin argument to define if operations are done by rows (margin = 1) or by columns (margin = 2)

apply(mat1, 1, sum)
##  V1  V2  V3 
##  15 520 250
apply(mat1, 2, sum)
## [1] 151 154 157 160 163

Using apply() one can apply (pun intended) any function by rows or columns

apply(mat1, 1, max)
##  V1  V2  V3 
##   5 108  50
apply(mat1, 2, mean)
## [1] 50.33333 51.33333 52.33333 53.33333 54.33333

NOTE: apply() is member of a family of functions that also includes: - tapply() for applying a function to an array by a grouping factor - lapply() and sapply() for applying a function to parts of a list (see next lecture) All these functions are EXTREMELY useful when an operation needs to be repeated across elements of a matrix, array or list. Their use and behaviour is a bit complex to cover in this course, but keep them in mind whenever you are trying to apply a function to specific margins of your data.
